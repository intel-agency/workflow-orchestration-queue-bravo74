## Project Purpose

OS-APOW (workflow-orchestration-queue) is a headless agentic orchestration platform that:
- Transforms GitHub Issues into execution orders fulfilled by AI agents
- Implements a 4-pillar architecture: Ear (Notifier), State (Queue), Brain (Sentinel), Hands (Worker)
- Provides "Zero-Touch Construction" - open an issue, get a PR

## Key Components

### 1. WorkItem Model (`src/models/work_item.py`)

Unified data structure for tasks:
- `WorkItem`: Task representation with id, issue_number, status, etc.
- `TaskType`: Enum (PLAN, IMPLEMENT, BUGFIX, REVIEW)
- `WorkItemStatus`: Enum mapping to GitHub labels
- `scrub_secrets()`: Credential sanitization utility

### 2. GitHub Queue (`src/queue/github_queue.py`)

Implements `ITaskQueue` interface:
- `fetch_queued_items()`: Get tasks with `agent:queued` label
- `claim_task()`: Assign-then-verify pattern for locking
- `update_status()`: Manage label transitions
- `post_comment()`: Post sanitized comments

### 3. Sentinel Orchestrator (`src/orchestrator_sentinel.py`)

Background service that:
- Polls GitHub Issues every 60 seconds
- Claims tasks using assign-then-verify
- Executes shell bridge commands
- Posts heartbeat comments every 5 minutes
- Handles graceful shutdown

### 4. Notifier Service (`src/notifier_service.py`)

FastAPI webhook receiver that:
- Validates HMAC signatures
- Triage issues based on content
- Queues tasks for processing
- Returns 202 Accepted within 10s

## Security Notes

- All webhook payloads must have valid HMAC SHA256 signatures
- Credentials are scrubbed before posting to GitHub
- Worker containers are network-isolated
- Resource limits: 2 CPUs, 4GB RAM per container
